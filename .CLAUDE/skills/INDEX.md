# Agent Skills Index

Skills live in `.agents/skills/<skill-name>/SKILL.md`. When a task matches a skill description, read that skill's `SKILL.md` before acting. Load only the skill files needed for the current task.

## Core Workflow

- `using-agent-skills`: Discover and invoke the right skill for the task.
- `context-engineering`: Configure project context, rules files, and session setup.
- `git-workflow-and-versioning`: Use disciplined git practices for code changes, commits, branches, and reviews.
- `planning-and-task-breakdown`: Break clear requirements into ordered implementation tasks.
- `spec-driven-development`: Create specs before significant new features or ambiguous work.
- `incremental-implementation`: Deliver multi-file or larger changes in small verified increments.
- `test-driven-development`: Drive logic changes, fixes, and behavior changes with tests.
- `debugging-and-error-recovery`: Diagnose failing tests, broken builds, and unexpected behavior from root cause.
- `doubt-driven-development`: Re-check non-trivial decisions when correctness risk is high.
- `code-review-and-quality`: Review changes for bugs, regressions, maintainability, and missing tests.

## Product and Frontend

- `frontend-ui-engineering`: Build or modify production-quality user-facing interfaces.
- `bm-design-system`: Apply Yvy Capital design tokens, visual identity, typography, and component patterns.
- `browser-testing-with-devtools`: Verify browser behavior, DOM, console, network, and visual output.
- `performance-optimization`: Profile and improve application performance.

## Architecture and Interfaces

- `api-and-interface-design`: Design stable APIs, module contracts, REST/GraphQL endpoints, and frontend/backend boundaries.
- `code-simplification`: Refactor working code for clarity without changing behavior.
- `deprecation-and-migration`: Remove, migrate, or sunset old systems and APIs safely.
- `source-driven-development`: Ground framework/library implementation details in authoritative documentation.

## Security, Compliance, and Delivery

- `security-and-hardening`: Harden code that handles user input, auth, sessions, storage, or integrations.
- `ci-cd-and-automation`: Configure build, test, deployment, and quality automation.
- `shipping-and-launch`: Prepare production releases, monitoring, rollout, and rollback plans.
- `documentation-and-adrs`: Record durable technical decisions and project documentation.

## Discovery and Product Thinking

- `interview-me`: Clarify underspecified requests one question at a time.
- `idea-refine`: Stress-test and refine raw ideas before planning or implementation.
